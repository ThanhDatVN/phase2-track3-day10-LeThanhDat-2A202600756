6. Request path và guardrails

Luồng này giải thích tại sao hệ thống giảm lỗi dây chuyền.

1
User prompt Nhận query thường, query dated và query privacy-sensitive.
2
Cache guard Bỏ qua query nhạy cảm, dùng TTL, n-gram cosine và false-hit check theo số/năm.
3
Circuit breaker CLOSED -> OPEN -> HALF_OPEN -> CLOSED, tránh retry storm khi provider lỗi.
4
Fallback provider Primary lỗi thì chuyển backup; route được ghi rõ để quan sát hành vi.
5
Static fallback Đường lui cuối cùng khi toàn bộ provider không khả dụng.

7. Phương pháp cải thiện tiếp theo

Backlog được xếp theo tác động lên SLO.

Priority 1

Giảm static fallback

Các SLO chính đã đạt, nhưng vẫn còn 1 static fallback. Cần đo route-level failure, tăng backup capacity hoặc thêm tertiary fallback rẻ để degraded path về 0.

Priority 2

Tune breaker theo scenario

Thử ma trận failure_threshold/reset_timeout để giảm circuit_open_count mà không làm recovery chậm. Report nên ghi cấu hình thắng dựa trên chaos result, không dựa vào cảm tính.

Priority 3

Nâng cấp observability

Bổ sung route labels, provider labels, cache decision logs và correlation id để sau này map metrics sang traces/logs theo chuẩn observability.
